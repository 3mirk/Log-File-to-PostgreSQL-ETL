Lightweight Python service that watches a folder for automatically generated LDS log files, parses route scan data, and loads the results into a PostgreSQL table for reporting and analytics.

## Features

- Monitors a configured directory for new log files
- Backfills existing unprocessed files on startup
- Pivots 5+ semi-structured log files into a consolidated single-row PostgreSQL schema
- Converts scan timestamps into PostgreSQL-friendly datetime values
- Inserts parsed records into PostgreSQL
- Uses a composite key to avoid duplicate loads

## Example input

```text
REQ=LDS
JOB=123456
STATUS=0;OK
DO=B
JOBRTE=B;SBLK;;;;ART BLOCKER 2;20250326211200;;
JOBRTE=B;GEN;;;;VFT ORBIT 8;20250326213300;;
JOBRTE=B;POL;;;;POLISHER 9;20250326214300;;
JOBRTE=B;ENG;;;;ENGRAVER 1;20250326215700;;
JOBRTE=B;;;;;ART DEBLOCKER 2;20250326220100;;
```

## Output columns

| Column | Description |
|---|---|
| `file_name` | Source log file name |
| `file_modified_at` | File modified timestamp |
| `job` | Job number |
| `do_val` | Eye / DO value from the file |
| `blk`, `blk_scan`, `blk_do` | Blocking station, scan time, and DO |
| `gen`, `gen_scan`, `gen_do` | Generator station, scan time, and DO |
| `pol`, `pol_scan`, `pol_do` | Polisher station, scan time, and DO |
| `eng`, `eng_scan`, `eng_do` | Engraver station, scan time, and DO |
| `dba`, `dba_scan`, `dba_do` | Deblocker station, scan time, and DO |

## Setup

### 1. Clone the repo

```bash
git clone https://github.com/YOUR_USERNAME/lds-log-etl.git
cd lds-log-etl
```

### 2. Create a virtual environment

```bash
python -m venv .venv
.venv\Scripts\activate
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

### 4. Create the PostgreSQL table

Run:

```sql
sql/create_job_entries.sql
```

in pgAdmin or your preferred PostgreSQL client.

### 5. Configure environment variables

Copy `.env.example` to `.env` and update the values:

```bash
copy .env.example .env
```

Example:

```env
DB_HOST=127.0.0.1
DB_PORT=5432
DB_NAME=che_dvi_track
DB_USER=postgres
DB_PASSWORD=postgres
MONITORED_FOLDER=I:\VISION\LDS\jobrte
```

## Run

```bash
python -m src.lds_log_etl.app
```

On startup, the app processes existing files that are not already in the database. After that, it continues watching the folder for newly created files.

## Notes

- Files with `.txt` extension or no extension are processed.
- Duplicate prevention is handled by the PostgreSQL primary key on `file_name` and `file_modified_at`.
- This project is designed for internal manufacturing analytics pipelines where log files are continuously generated by lab systems.
